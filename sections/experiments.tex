\section{Experimental Evaluation}

This section presents the experimental evaluation of ADRDS, focusing on the effectiveness of the detection pipeline, the responsiveness of the system, and its suitability for interactive demonstration and exploratory analysis. The experiments are designed to validate both detection accuracy and real-time performance under controlled and reproducible conditions.

\subsection{Experimental Setup}

All experiments are conducted within a sandboxed environment using synthetic and benchmark-derived behavioral data. Ransomware-like behaviors are generated using the behavioral simulation engine described earlier, while benign activity is simulated to represent normal system workloads. The detection engine processes streaming behavioral features in real time, and all results are visualized through the interactive dashboard.

Supervised learning models are trained using labeled samples derived from the BigMalware dataset and selected subsets of the EMBER dataset. Unsupervised anomaly detection models are trained exclusively on benign behavior traces to establish a baseline representation of normal system activity. The trained models are then evaluated against simulated ransomware scenarios that vary in intensity and behavioral characteristics.

\subsection{Evaluation Metrics}

Detection performance is assessed using standard classification metrics, including accuracy, precision, recall, and F$_1$-score for supervised models. For anomaly detection, reconstruction error distributions and threshold-based true positive and false positive rates are analyzed. In addition to detection accuracy, system responsiveness is evaluated by measuring end-to-end processing latency, defined as the time between behavioral event generation and detection output visualization.

\subsection{Detection Performance}

Experimental results demonstrate that the supervised classification models achieve high accuracy in identifying known ransomware behavior patterns, while the unsupervised anomaly detection component effectively flags deviations associated with previously unseen attack scenarios. The hybrid risk fusion mechanism improves overall robustness by reducing false positives and enabling earlier detection during pre-encryption stages.

Across multiple simulated attack variants, ADRDS consistently detects malicious behavior before the onset of sustained encryption activity. This early detection capability highlights the advantage of combining stage-aware behavioral features with hybrid machine learning models.

\subsection{Real-Time Responsiveness}

ADRDS maintains near real-time performance throughout all experimental scenarios. The end-to-end detection pipeline operates with sub-second latency, ensuring that detection outcomes and visualization updates occur promptly as behavioral events unfold. This responsiveness is critical for interactive demonstration and supports practical exploration of detection thresholds and behavioral variations.

\subsection{Ablation and Sensitivity Analysis}

To assess the contribution of individual system components, ablation experiments are performed by selectively disabling elements of the detection pipeline. Results indicate that removing the anomaly detection component increases false negatives for novel attack scenarios, while disabling risk fusion leads to higher false positive rates. Sensitivity analysis of detection thresholds further illustrates the trade-off between detection sensitivity and alert stability.

\subsection{Discussion}

The experimental evaluation confirms that ADRDS effectively balances detection accuracy, interpretability, and real-time performance. While the experiments focus on simulated ransomware behavior, the use of benchmark datasets for model training and structured behavioral taxonomy enhances the generalizability of the findings. These results demonstrate that ADRDS is well-suited for both educational demonstrations and exploratory ransomware detection research.
