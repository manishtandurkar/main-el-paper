\section{Demo Scenario and Walkthrough}

ADRDS is designed as an interactive demonstration platform that allows users to observe, analyze, and understand ransomware detection in real time. The demo scenario showcases the end-to-end operation of the system, from behavior simulation to detection and response, emphasizing transparency and user-driven exploration.

\subsection{Demo Setup}

The demonstration is conducted within a controlled and sandboxed environment to ensure safety and reproducibility. At the start of the demo, the system is initialized in a benign state, where normal application behavior is simulated and monitored by the ADRDS pipeline. All detection thresholds are set to conservative default values to reflect a realistic deployment baseline.

The visualization dashboard presents the initial system state, including baseline behavioral metrics, zero or low risk scores, and inactive alert indicators.

\subsection{Scenario Execution}

The demo proceeds through a sequence of progressively malicious scenarios that align with the behavioral taxonomy described earlier.

\textbf{Stage 1: Benign Baseline Behavior.}  
The simulator generates normal workload activity such as routine file access and process execution. The detection engine classifies this activity as benign, and the dashboard displays stable behavioral metrics with low risk scores.

\textbf{Stage 2: Pre-Encryption Ransomware Activity.}  
Ransomware-like behaviors are introduced, including rapid file enumeration and increased process activity. Although no destructive actions occur at this stage, the anomaly detection component begins to register deviations from baseline behavior, resulting in a gradual increase in the system risk score.

\textbf{Stage 3: Active Encryption Simulation.}  
The simulator escalates to high-intensity behaviors characteristic of ransomware encryption, such as burst file writes and sharp entropy variations. The supervised classifier and anomaly detector jointly flag the activity as malicious, causing the fused risk score to exceed the alert threshold.

\subsection{Detection and Response Visualization}

As malicious behavior intensifies, the visualization layer updates in real time to reflect attack progression and detection outcomes. Attack stages are highlighted on a temporal timeline, while risk scores and confidence values are dynamically updated.

Once the detection threshold is crossed, ADRDS triggers simulated containment actions, such as process isolation and alert generation. These actions are visualized immediately, allowing observers to correlate behavioral changes with detection decisions and response outcomes.

\subsection{Interactive Exploration}

A key feature of the demo is its interactive nature. Users can adjust simulation parameters, modify detection thresholds, and replay scenarios to explore how changes in attacker behavior or sensitivity settings impact detection performance. This capability enables hands-on experimentation and fosters intuitive understanding of ransomware behavior, detection trade-offs, and system responsiveness.

\subsection{Demonstration Value}

The demo walkthrough illustrates ADRDS’s ability to provide early-stage ransomware detection, transparent decision-making, and real-time feedback within a safe environment. By combining behavioral simulation, hybrid machine learning, and interactive visualization, ADRDS serves as both an educational tool and an experimental platform for evaluating ransomware detection strategies.
