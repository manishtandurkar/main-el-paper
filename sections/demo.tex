\section{Demonstration Scenario and Walkthrough}

This section describes a live demonstration designed to illustrate the real-time detection capabilities and transparent feedback mechanisms of ADRDS.

\subsection{Scenario Configuration}

The demonstration is initiated within a sandboxed environment, with ADRDS calibrated to a verifiable benign state. The simulator generates standard background activity, including routine file access and process management, while the detection engine monitors the system. All detection thresholds are initialized to conservative default values, with the dashboard displaying nominal risk metrics.

\subsection{Demonstration Phases}

The scenario proceeds through three distinct phases, each representing an escalation in the simulated threat level.

During the initial baseline operation phase, the simulator executes a typical user workload consisting of document operations and application launches. The detection engine classifies these activities as benign, maintaining low risk scores, thereby establishing a baseline for normal operational behavior.

The second phase introduces pre-encryption indicators as simulation intensity is increased without actual file encryption. This phase exhibits early warning behaviors such as rapid directory traversal, elevated file enumeration, and minor process anomalies. Although no encryption occurs, the anomaly detector identifies deviations from the baseline, causing risk scores to incrementally rise while the temporal timeline reflects this shift in activity.

The third phase initiates automated attack simulation as the simulator transitions to the encryption phase, characterized by high-velocity file write operations, significant entropy spikes, and rapid file renaming. Both the supervised classifier and the anomaly detector are triggered, causing the cumulative risk score to exceed the alert threshold. Consequently, the dashboard issues a warning and ADRDS initiates simulated containment protocols.

\subsection{Real-Time Visualization}

Throughout the demonstration, the visualization layer updates continuously. Observers can monitor the progression of attack stages on a temporal timeline, observe risk scores and confidence values increasing with suspicious events, and identify the precise moment thresholds are breached. Additionally, simulated response actions, such as process isolation and alert logging, are displayed. This transparency allows for the correlation of specific behavioral events with detection outcomes.

\subsection{Interactive Elements}

ADRDS emphasizes interactivity, allowing users to modify simulation intensity, adjust detection thresholds, and replay scenarios to observe varied outcomes. This capability facilitates immediate inquiry into system sensitivity and performance under different configurations, replacing abstract theoretical discussion with empirical observation.

\subsection{Summary}

The demonstration effectively highlights the system's capacity for early-stage detection, transparent decision-making, and responsive feedback within a secure environment. It underscores the efficacy of combining behavioral simulation, hybrid machine learning, and interactive visualization to provide a rigorous yet accessible approach to ransomware detection.
