\section{Behavioral Taxonomy}

Catching ransomware early requires understanding how it behaves over time. Rather than hunting for static signatures, we structure detection around a taxonomy of observable behaviors that unfold across distinct attack stages. This section lays out that taxonomy and explains how it feeds into feature extraction and detection logic.

\subsection{Stages of a Ransomware Attack}

We model ransomware as a sequence of stages, each defined by the host-level events it produces. Recognizing these stages helps ADRDS detect partial attacks and estimate how far an intrusion has progressed.

\subsubsection{Stage 1: Reconnaissance and Setup}
In the opening moments, ransomware probes its environment. It might enumerate directories, query system information, or check for virtual machine artifacts that could indicate a sandbox. These actions are subtle, but they set the stage for what comes next.

\subsubsection{Stage 2: Privilege Escalation and Persistence}
Once reconnaissance wraps up, attackers often try to strengthen their grip. We see abnormal process spawning, modifications to startup entries, and interactions with system-level resources. These moves expand access and help the malware survive reboots.

\subsubsection{Stage 3: File Enumeration and Targeting}
Before encrypting anything, ransomware identifies its targets. This stage features high-volume read operations, rapid directory traversal, and selective filtering based on file extensions or folder paths. The burst of file activity stands out sharply against typical user workloads.

\subsubsection{Stage 4: Encryption and Destruction}
Here the damage happens. Intensive file writes, sudden renames, and sharp entropy spikes mark the encryption phase. Files transform from readable documents into scrambled blobs at a pace that no ordinary application would match.

\subsubsection{Stage 5: Communication and Wrap-Up}
Some ransomware phones home after encryption—exchanging keys, reporting success, or awaiting further instructions. We watch for anomalous network connections and communication patterns that coincide with the encryption window.

Each stage translates into measurable indicators that ADRDS extracts and monitors. These include file system metrics such as access frequency, modification rate, and entropy changes; process-level attributes like CPU load, spawning patterns, and API call rates; temporal patterns such as burstiness and sustained high-activity windows; and stage context indicating which phase the current activity aligns with. By correlating low-level telemetry with high-level attack stages, the system can identify suspicious behavior and attribute it to a specific lifecycle phase.

\subsection{Why This Taxonomy Matters}

The behavioral taxonomy is not just academic scaffolding—it shapes everything ADRDS does. It guides how we extract features, what the ML models learn, and how the visualization layer presents results. Because detection is stage-aware, ADRDS can catch ransomware during early reconnaissance or file enumeration, often before any real damage occurs. Equally important, linking alerts to specific stages makes outputs interpretable, giving analysts and students concrete context instead of opaque scores.
