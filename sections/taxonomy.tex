\section{Behavioral Taxonomy}

Effective ransomware detection requires a structured understanding of how malicious behavior manifests over time at the host level. Rather than relying on static signatures, ADRDS adopts a behavior-centric taxonomy that decomposes ransomware activity into distinct, observable stages. This taxonomy serves as the foundation for feature extraction, detection logic, and interpretability within the ADRDS framework.

\subsection{Ransomware Lifecycle Stages}

ADRDS models ransomware behavior as a sequence of stages that reflect common attack workflows observed across modern ransomware families. Each stage is characterized by a set of observable host-level behaviors that can be monitored in real time.

\subsubsection{Stage 1: Reconnaissance and Environment Preparation}
In the initial stage, ransomware performs lightweight reconnaissance to assess the execution environment. Observable behaviors include directory enumeration, system information queries, and checks for virtualization or debugging artifacts. While these actions are often subtle, they establish a baseline for subsequent malicious activity.

\subsubsection{Stage 2: Privilege Escalation and Persistence}
Following reconnaissance, ransomware may attempt to elevate privileges or establish persistence mechanisms. This stage is characterized by behaviors such as abnormal process spawning, modification of startup configurations, and interactions with system-level resources. These actions increase the potential impact of later stages by expanding access and execution longevity.

\subsubsection{Stage 3: File Enumeration and Target Selection}
Before encryption, ransomware typically performs extensive file system traversal to identify target files and directories. This stage exhibits high-volume read operations, rapid directory walking, and selective filtering based on file extensions or directory paths. Such access patterns often deviate significantly from benign application behavior.

\subsubsection{Stage 4: Encryption and Destructive Activity}
The encryption stage represents the most destructive phase of a ransomware attack. It is characterized by intensive file write operations, rapid file renaming, and sharp increases in file entropy due to cryptographic transformations. These behaviors occur at a scale and speed that distinguish ransomware from most legitimate applications.

\subsubsection{Stage 5: Communication and Post-Encryption Actions}
In some cases, ransomware performs outbound communication to exchange encryption keys, report execution status, or receive additional instructions. Observable indicators include anomalous network connections, unusual protocol usage, and communication patterns that coincide temporally with encryption activity.

\subsection{Behavioral Feature Mapping}

Each lifecycle stage is mapped to a set of behavioral indicators that are extracted and quantified by ADRDS. These indicators include, but are not limited to:
\begin{itemize}
    \item File system metrics such as access frequency, modification rate, and entropy variation
    \item Process-level attributes including CPU utilization, process spawning patterns, and API invocation rates
    \item Temporal characteristics such as burstiness and sustained high-activity intervals
    \item Contextual stage information derived from lifecycle progression
\end{itemize}

By associating low-level telemetry with higher-level attack stages, ADRDS enables both accurate detection and meaningful interpretation of alerts.

\subsection{Role of the Taxonomy in ADRDS}

The behavioral taxonomy plays a central role in the ADRDS architecture. It guides feature extraction, informs the machine learning detection engine, and provides semantic context for visualization and user interaction. Unlike monolithic detection approaches, this structured taxonomy allows ADRDS to reason about partial attack progression, enabling early-stage detection and stage-aware risk assessment. Additionally, the taxonomy enhances explainability by linking detection outcomes to specific behavioral stages, improving transparency for analysts and educators.
